## 路由配置规范

uni-app 使用 `pages.json` 文件进行路由配置，以下是配置规范与最佳实践：

### 1. 页面路径规范
- **路径格式**: 必须使用 `pages/` 开头，例如 `pages/index/index`
- **目录结构**: 每个页面应放在独立的目录中，目录名与页面名一致
- **文件命名**: Vue 文件使用小写字母，多个单词使用连字符 `-` 分隔（如 `user-center.vue`）

### 2. pages.json 结构说明
```json
{
  "pages": [
    {
      "path": "pages/index/index",        // 页面路径
      "style": {
        "navigationBarTitleText": "首页",   // 导航栏标题
        "navigationStyle": "custom"        // 自定义导航栏
      }
    }
  ],
  "globalStyle": { ... },                  // 全局样式
  "tabBar": { ... },                       // 底部导航栏
  "subPackages": [ ... ]                   // 分包配置
}
```

### 3. 页面样式配置
- **navigationBarTitleText**: 页面标题，应简洁明了
- **navigationStyle**: 通常设为 `"custom"` 以使用自定义导航栏
- **pageOrientation**: 视频播放等页面可设为 `"auto"` 支持横竖屏切换
- **backgroundColor**: 页面背景色，建议与主题一致

### 4. 底部导航栏 (tabBar) 配置
- **color**: 未选中颜色，建议使用灰色
- **selectedColor**: 选中颜色，使用主题色 `#2ff0ff`
- **backgroundColor**: 背景色，建议使用半透明磨砂效果
- **list**: 最多5个，每个需包含 `pagePath`（页面路径）和 `text`（显示文本）

### 5. 分包加载配置
- **使用场景**: 当页面较多时，使用分包优化首屏加载
- **配置字段**: `subPackages` 或 `subpackages`
- **目录结构**: 每个分包独立目录，root 字段指定分包根目录
- **页面引用**: 分包内页面路径从分包根目录开始

### 6. 动态导入与代码分割
- **⚠️ 重要**: uni-app 小程序环境不支持标准的 `import()` 动态导入语法
- **替代方案**:
  - 使用条件编译 `#ifdef`、`#ifndef` 按平台加载
  - 使用分包加载机制实现代码分割
  - 使用 `uni.requireModule` 加载原生模块
- **检查工具**: 运行 `npm run check-config` 检测项目中的动态导入

### 7. 常见错误预防
1. **路径错误**: 确保 `pages.json` 中的路径与文件系统实际路径一致
2. **重复路径**: 同一路径不能在 `pages` 和 `subPackages` 中重复出现
3. **分包限制**: 分包不能嵌套，主包与分包页面不能互相引用
4. **动态导入**: 避免使用 `import()`，改用静态导入或条件编译

### 8. 配置检查清单
✅ 运行 `npm run check-config` 验证配置
✅ 所有页面路径均以 `pages/` 开头  
✅ 页面标题与功能相符
✅ 底部导航栏选中颜色为主题色
✅ 分包配置合理，主包体积控制在 2MB 以内
✅ 无动态导入语法（小程序环境）

## 动态导入错误预防

uni-app 小程序环境对 JavaScript 动态导入 (`import()`) 支持有限，不当使用会导致构建错误或运行时异常。以下是预防措施：

### 1. 问题描述
- **构建错误**: 在小程序构建时，`import()` 可能无法正确解析，导致打包失败
- **运行时错误**: 小程序平台可能不支持动态加载，导致 `Promise` 拒绝
- **代码分割失效**: 预期的代码分割效果无法实现，增加首包体积

### 2. 检测方法
- **使用检查脚本**: 运行 `npm run check-config` 扫描项目中的动态导入
- **构建验证**: 在微信开发者工具中构建项目，观察控制台报错
- **代码审查**: 人工检查 `pages/`、`components/`、`utils/` 目录下的 `.vue` 和 `.js` 文件

### 3. 预防措施
1. **避免使用 `import()`**: 在小程序相关代码中完全避免动态导入语法
2. **静态导入替代**: 将 `import()` 改为静态 `import` 语句
3. **条件编译**: 使用 `#ifdef`、`#ifndef` 按平台加载模块
   ```javascript
   // 正确：条件编译
   #ifdef H5
   import { someModule } from 'some-path'
   #endif
   ```
4. **分包加载**: 使用 `pages.json` 的 `subPackages` 配置实现代码分割
5. **异步组件**: Vue 3 的 `defineAsyncComponent` 在小程序环境中需谨慎使用

### 4. 已遇到错误的解决方案
如果已出现动态导入相关错误，按以下步骤修复：

1. **定位问题文件**: 根据错误信息找到使用 `import()` 的文件
2. **替换为静态导入**: 将 `import('module')` 改为 `import module from 'module'`
3. **条件加载**: 如需按条件加载，使用 `require`（仅限 Node.js 环境）或平台判断
4. **验证修复**: 重新运行 `npm run check-config` 和 `npm run build`

### 5. 构建配置检查
确保 `vite.config.js` 正确配置了 uni-app 插件：
```javascript
import { defineConfig } from 'vite'
import uni from '@dcloudio/vite-plugin-uni'

export default defineConfig({
  plugins: [uni()],
  // 其他配置...
})
```

### 6. 开发规范
- **新代码审查**: 提交代码前运行检查脚本
- **团队培训**: 确保所有开发者了解小程序动态导入限制
- **文档维护**: 将本预防措施纳入项目开发文档

### 7. 应急处理
如线上版本因动态导入出错，立即：
1. 回滚到上一个稳定版本
2. 使用检查脚本定位问题
3. 发布热修复版本
